Guia de Despliegue — GitHub Pages

Cada caso se publica en un repositorio dedicado de GitHub Pages. Solo se sube el sitio HTML con capacidad de exportar PDF. Nada mas.

Prerequisitos

  • Git instalado y autenticado con GitHub
  • Python 3 con pip install markdown
  • Repositorio dedicado creado en GitHub (puede estar vacio)

Paso a Paso

1. Crear repositorio dedicado en GitHub

  • Ir a github.com → New Repository
  • Nombre sugerido: ka-{agente}-{caso} (ej: ka-trib-airbnbvilla2026)
  • Puede ser publico o privado (privado requiere GitHub Pro para Pages)
  • NO inicializar con README (el script lo crea)

2. Ejecutar el script de deploy

bash tools/deploy-gh-pages.sh <carpeta_caso> <repo_url>

Ejemplo:

bash tools/deploy-gh-pages.sh airbnb_villarrica_202602 https://github.com/Brownbull/ka-trib-airbnbvilla2026.git

El script automaticamente: 1. Genera el sitio HTML desde los Markdown del caso 2. Clona el repo dedicado 3. Copia el sitio generado (HTML + CSS + Mermaid) 4. Hace commit y push

3. Activar GitHub Pages

  1. Ir al repositorio dedicado en GitHub
  2. Settings → Pages
  3. Source: Deploy from a branch
  4. Branch: main / / (root)
  5. Guardar
  6. Esperar ~1 minuto — el sitio estara en https://<usuario>.github.io/<repo>/

Actualizar un sitio existente

Despues de modificar documentos del caso, ejecutar el mismo comando:

bash tools/deploy-gh-pages.sh <carpeta_caso> <repo_url>

El script detecta el repo existente, actualiza los archivos y hace push.

Que se sube al repo dedicado

repo-dedicado/
├── index.html              ← PANEL.md (landing page)
├── assets/
│   └── style.css           ← Estilos del sitio
├── deploy-guide.html       ← Esta guia en formato HTML
├── .nojekyll               ← Evita procesamiento Jekyll
├── README.md               ← Generado automaticamente
├── 99_entregable/
│   ├── PANEL.html
│   ├── 01_critico/...
│   ├── 02_bloqueantes/...
│   └── ...
├── CASE.html
└── ... (todas las paginas del caso)

Que NO se sube

  • Codigo fuente de agentes (YAML, sidecar)
  • Scripts de build
  • Otros casos
  • Archivos de la plataforma (CLAUDE.md, SERVICE-CONFIG.yaml)

Funcionalidades del sitio

  • Sidebar de navegacion con todos los documentos del caso
  • Diagramas Mermaid renderizados (estados, flujos, timelines)
  • Boton "Exportar PDF" en cada pagina (usa el dialogo de impresion del navegador)
  • Tags de confianza estilizados ([VERIFICADO-WEB], [ANALISIS-IA], etc.)
  • Disclaimers resaltados en amarillo
  • Responsive — se adapta a movil (sidebar se oculta)
  • Print-friendly — CSS especial para impresion/PDF
